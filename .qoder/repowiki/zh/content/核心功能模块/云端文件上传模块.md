# 云端文件上传模块

## 概述

“云端文件上传模块”是应用内负责将本地文件高速、稳定地上传至第三方对象存储（目前集成七牛云）的核心模块。为解决 Electron 应用中处理 GB 级大文件带来的内存溢出与 UI 阻塞问题，本模块在架构上采用了“主渲染分离”的最佳实践。

## 核心架构设计

### 1. Node.js 主进程流式上传 (Backend)

所有的文件 I/O 与网络请求全部由主进程承担，绕过浏览器的 Chromium 网络栈：

- **`src/main/upload.ts`**：
  - 基于 Node.js 原生的 `http/https` 与 `fs.createReadStream` 实现。
  - **小文件极速表单上传**：4MB 以下文件走直接的表单提交流程。
  - **大文件分片断点续传**：4MB 以上文件调用 SDK 的分片策略，将进度记录持久化到临时目录（`os.tmpdir()`），支持续传。
  - **网络异常轮询重试**：实现了一套带指数退避（Exponential Backoff）的重试机制。在触发网络断开（如 `ENOTFOUND`, `ECONNRESET`）时，自动挂起并轮询检测网络连通性，待网络恢复后自动继续上传，保证了弱网环境下的极高稳定性。

### 2. Vue 渲染进程与 IPC (Frontend)

- **`src/renderer/src/views/FileUpload/index.vue`**：提供极简的拖拽交互界面，智能推断文件类型（如 `.mp4`, `.png`）并感知文件大小（超过 4MB 自动打上“大文件模式”标签）。
- **进度透传**：通过 `window.api.file.onUploadProgress` 实时接收主进程每秒发来的上传百分比，实现丝滑的进度条更新。
- **跨域与缓存处理**：
  - 在 `src/main/index.ts` 中设置了 `webSecurity: false`，并在 `index.html` 调整了 CSP，彻底解除了图片展示和网络请求的同源拦截。
  - 前端获取到上传成功的新 URL 后，自动拼接 `?t=时间戳` 绕过浏览器强制缓存，确保头像等资源立即刷新。

## 组件化与复用

- 模块依赖于前面封装的 `useFormat` (处理文件大小展现) 和统一样式文件 `tool-layout.scss`，代码高度内聚，便于后续扩展如“阿里云 OSS”、“AWS OSS”等多云存储支持。
